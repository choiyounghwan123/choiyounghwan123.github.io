## 1. What is DataBase?
**데이터베이스(Database)**는 데이터를 체계적으로 저장하고 관리할 수 있도록 한 구조화된 저장소를 말합니다. 
줄여서 DB라고 많이 부릅니다.

### 데이터베이스 시스템 도입 배경
데이터베이스를 도입하기 전에는 파일시스템이라는것을 사용하였습니다. 하지만, 파일시스템은 데이터 무결성, 종속성등 많은 문제점들을 가지고 있었습니다. 이런 문제점들을 해결하기 위해서 도입된 것이 데이터베이스입니다.

**파일시스템은 독립된 파일 단위로 데이터를 저장하므로, 데이터 중복성과 데이터 종속성이 발생합니다.**


### 데이터베이스의 주요 특징
- 공유데이터: 특정 조직의 여러 사용자가 함께 소유하고 이용할 수 있는 공유 데이터
- 통합데이터: 최소의 중복과 통제 가능한 중복만 허용한 데이터
- 저장 데이터: 컴퓨터가 접근할 수 있는 매체에 저장된 데이터
- 운영데이터: 조직의 주요 기능을 수행하기 위해 지속적으로 꼭 필요한 데이터
- 실시간 접근: 사용자의 데이터 요구에 실시간으로 응답
- 계속 변화: 데이터의 계속적인 삽입,삭제, 수정을 통해 현재의 정확한 데이터를 유지
- 동시 공유: 서로 다른 데이터의 동시 사용 뿐만 아니라 같은 데이터의 동시 사용도 지원
- 내용 기반 참조: 데이터가 저장된 주소나 위치가 아닌 내용으로 참조

### 데이터베이스의 필요성 및 장점
- 데이터 무결성 유지: 데이터베이스가 데이터의 일관성을 유지하는 방법, 무결성 제약 조건
        
## 2. What is DBMS?
**데이터베이스(Database)**는 데이터를 저장하는 저장소 일뿐이고, DataBase를 실질적으로 관리하는 소프트웨어를 **DBMS(DataBase Management System)** 라고합니다.

DBMS는 데이터를 저장, 관리, 검색, 수정하는데 사용되고, 데이터를 효율적으로 관리하고 여러 사용자들이 동시에 데이터를 접근하고 수정할 수 있습니다.

### DBMS의 역활과 기능
- 1. **데이터 정의(Definition)**
    - **스키마 정의:** DBMS는 데이터베이스에 저장될 데이터의 구조를 정의합니다. 예를 들어, 테이블, 열, 데이터 유형, 제약 조건 등을 설정하여 데이터의 저장 형태를 설계합니다.
    - **데이터 제약 조건 설정:** 각 열에 들어가는 데이터 값의 유효성을 확인하고, 기본 키(Primary Key), 외래 키(Foreign Key) 등의 무결성 제약을 설정해 데이터의 일관성을 보장합니다.
- 2. **데이터베이스 구축 및 저장(Storage)**
    - **데이터 저장소 생성:** DBMS는 물리적 스토리지에서 데이터를 효율적으로 관리할 수 있도록 데이터를 저장하고, 저장소를 생성합니다.
    - **인덱스 관리**: DBMS는 데이터를 빠르게 검색할 수 있도록 테이블에 인덱스를 생성하고 관리합니다.
- 3. **데이터 조작(Manipulation)**
    - **CRUD기능 제공:** DBMS는 데이터를 **생성(Create), 읽기(Read), 수정(Update), 삭제(Delete)**할 수 있는 기능을 제공합니다. 이를 통해 사용자는 SQL같은 쿼리 언어를 사용하여 데이터베이스를 이용할 수 있습니다.
    - **데이터 검색 최적화:** 다양한 최적화 기법을 통해 데이터를 빠르게 검색할 수 있는 도구를 제공하고, 복잡한 쿼리도 효율적으로 처리합니다.
- 4. **동시성 제어(Concurrency Control)**
    - **다중 사용자 지원:** DBMS는 여러 사용자가 동시에 동일한 데이터베이스에 접근할 수 있도록 동시성 제어를 제공합니다. 데이터 충돌을 방지하고 데이터의 일관성을 유지하기 위해 락(Lock)이나 타임스탬프 등의 기법을 사용합니다.
    - **트랜잭션 관리:** DBMS는 트랜잭션의 기능을 제공하여 다중 사용자 환경에서도 안전하게 데이터를 처리합니다. 트랜잭션은 **ACID**속성(원자성, 일관성, 고립성, 지속성)을 준수하여 데이터의 무결성을 보장합니다.
- 5. **데이터 보안(Security)**
    - **사용자 권한 관리:** DBMS는 사용자 인증 및 권한 관리를 통해 데이터 접근을 제어합니다. 특정 사용자는 데이터베이스에 대한 읽기, 쓰기, 수정등의 권한을 제한받을 수 있습니다.
    - **데이터 암호화:** 민감한 데이터를 암호화하여 보안을 강화하고, 데이터 손실, 해킹등도 보호합니다.
- 6. **백업 및 복구(Backup and Recovery)**
    - **백업:** DBMS는 시스템  오류나 데이터 손상에 대비해 정기적으로 데이터를 백업합니다. 이 백업을 통해 데이터 손실 시 복구 작업을 수행할 수 있습니다.
    - **복구:** DBMS는 장애 발생 시 데이터를 이전 상태로 복구할 수 있는 기능을 제공합니다. 트랜잭션 로그를 기반으로 장애 시점 전으로 데이터를 복구합니다.
- 7. **데이터 무결성 유지**
    - **무결성제약:** DBMS는 데이터의 무결성을 유지하기 위해 제약조건을 제공합니다. 데이터 무결성 제약에는 참조 무결성, 도메인 무결성, 엔티티 무결성 등이 있으며, 이를 통해 데이터의 일관성을 유지합니다.
